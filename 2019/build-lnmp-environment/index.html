Build Lnmp Environment

Written by Prometheus.Zhang with ♥ on August 8, 2019 in Build Environment

环境准备

安装CentOS7

安装nginx

  • 安装CentOS7时选择安装了基本开发环境，所以不必先安装gcc，gcc-c++

==1. 安装rewrite模块所需PCRE库==

cd /usr/loacl                        #将其装入local文件夹中
wget https://sourceforge.net/projects/pcre/files/pcre/8.43/pcre-8.43.tar.gz
#下载最新PCRE


tar -zxvf pcre-8.43.tar.gz #解压
cd pcre-8.36
./configure          #配置环境
make && make install #编译安装

==2. 安装SSL库==

和第一步原理类似

cd ..
wget http://www.openssl.org/source/openssl-1.1.1c.tar.gz
tar -zxvf openssl-1.0.1j.tar.gz
cd openssl-1.0.1j
./configure
make && make install

==3. 安装zlib库==

cd /usr/local/
wget http://zlib.net/zlib-1.2.11.tar.gz
tar -zxvf zlib-1.2.11.tar.gz
cd zlib-1.2.11
./configure
make && make instal

==4.安装nginx==

cd /usr/local/
wget http://nginx.org/download/nginx-1.16.0.tar.gz
tar -zxvf nginx-1.16.0.tar.gz
cd nginx-1.16.0 
./configure --user=nobody --group=nobody --prefix=/usr/local/nginx --with-http_stub_status_module --with-http_gzip_static_module --with-http_realip_module --with-http_sub_module --with-http_ssl_module
make && make install

安装成功后/usr/local/nginx/sbin/nginx命令报错

ldd /usr/local/nginx/sbin/nginx查看一下依赖

网上查找后我觉得之前将openssl装在了/usr/local里，所以应该建立软连接

再看一下依赖

之后再/usr/local/nginx/sbin/nginx启动成功！

安装PHP并配置nginx

cd /usr/local/
wget https://www.php.net/distributions/php-7.3.7.tar.gz
tar -zxvf php-7.3.7.tar.gz
cd php-7.3.7
./configure --prefix=/usr/local/php --with-config-file-path=/usr/local/php/etc --with-mcrypt=/usr/include --with-mysql=mysqlnd --with-mysqli=mysqlnd --with-pdo-mysql=mysqlnd --with-gd --with-iconv --with-zlib --enable-xml --enable-bcmath --enable-shmop --enable-sysvsem --enable-inline-optimization --enable-mbregex --enable-fpm --enable-mbstring --enable-ftp --with-openssl --enable-pcntl --enable-sockets --with-xmlrpc --enable-zip --enable-soap --without-pear --with-gettext --enable-session --with-curl --with-jpeg-dir --with-freetype-dir --enable-opcache #配置环境

提示libxml2未找到，选择直接yum安装它

yum install libxml2

显示已安装最新版本，网上查找提示再安装libxml2-devel

yum install libxml2-devel.x68_64

安装成功

然而又提示configure: error: Cannot find OpenSSL’s libraries

网上查得知尽管我们安装了openssl，但是查找库文件还是要去/usr/lib

find / -name libssl.so #搜索openssl库文件位置


ln -s /usr/local/lib64/libssl.so /usr.lib #建立软连接

继续，又爆出error: cURL version 7.15.5 or later is required to compile php with cURL support

yum install libcurl-devel.x86_64

error: jpeglib.h not found.

yum install libjpeg-turbo-devel.x86_64

error: png.h not found.

yum install libpng-devel.x86_64

error: freetype-config not found.

yum install freetype-devel.x86_64

error: Please reinstall the libzip distribution

将原有卸载选择源码安装

yum remove -y libzip
wget https://nih.at/libzip/libzip-1.2.0.tar.gz
tar -zxvf libzip-1.2.0.tar.gz.tar.gz
cd libzip-1.2.0/
./configure
make&&make install

随后./configure成功，make&&install

./configure --prefix=/usr/local/php --with-config-file-path=/usr/local/php/etc --with-mcrypt=/usr/include --with-mysql=mysqlnd --with-mysqli=mysqlnd --with-pdo-mysql=mysqlnd --with-gd --with-iconv --with-zlib --enable-xml --enable-bcmath --enable-shmop --enable-sysvsem --enable-inline-optimization --enable-mbregex --enable-fpm --enable-mbstring --enable-ftp --with-openssl --enable-pcntl --enable-sockets --with-xmlrpc --enable-zip --enable-soap --without-pear --with-gettext --enable-session --with-curl --with-jpeg-dir --with-freetype-dir --enable-opcache
make&&make install

致命错误：zipconf.h：没有那个文件或目录

find / -name zipconf.h

发现在/usr/local/libzip-1.2.0/lib里,复制到需要的地方

cp /usr/local/libzip-1.2.0/lib/zipconf.h /usr/local/include/zipconf.h

安装成功~

添加环境变量，将nginx和php加入/etc/profile中

更改配置文件名称，启动php-fpm

配置nginx解析php

vim /usr/local/nginx/conf/nginx.conf

设置用户和组，与php-fpm一致，添加默认文件index.php,更改php设置块。

重启测试一下

killall nginx
nginx

测试成功！最后将配置文件移动到指定目录中。

 cp /usr/local/php-7.3.7/php.ini-development /usr/local/php/etc/php.ini 
 #将配置文件移动到指定目录中

安装配置mysql

wget下载过慢，所以用迅雷在官网上下载后再移到虚拟机中

先创建用户和组

groupadd mysql
useradd -s /sbin/nologin -g mysql mysql

创建需要的文件夹并赋予权限

mkdir -p /usr/local/mysql/data
chown -R mysql:mysql /usr/local/mysql/
touch /var/log/mysql.log
chown -R mysql:mysql /var/log/mysql.log
mkdir -p /var/lib/mysql
chown -R mysql:mysql /var/lib/mysql/
mkdir -p /var/run/mysqld
chown -R mysql:mysql /var/run/mysqld/
mkdir /var/log/mariadb
touch /var/log/mariadb/mariadb.log
chown -R mysql:mysql /var/log/mariadb/ 
mkdir -p /var/run/mariadb
chown -R mysql:mysql /var/run/mariadb/

修改配置文件

初始化数据库，并安装秘钥

mysqld --defaults-file=/etc/my.cnf --initialize-insecure --basedir=/usr/local/mysql --datadir=/usr/local/mysql/data --user=mysql
mysql_ssl_rsa_setup

将usr/local/mysql/bin文件夹加入环境变量

mysqld_safe --user=mysql & #安全启动


mysql -u root#登录数据库

登录成功！

修改一下php.ini使其支持mysql

我们再建一个网页测试一下

测试成功，搭建完成！

将它们设置为开机自启动

cp /usr/local/php-7.3.7/sapi/fpm/php-fpm.service /usr/lib/systemd/system/
systemctl daemon-reload 
systemctl enable php-fpm.service

创建nginx.service，实现nginx自启动

systemctl daemon-reload 
systemctl enable nginx


cp -a /usr/local/mysql/support-files/mysql.server /etc/init.d/mysql
chkconfig --add mysql

设置完成。
